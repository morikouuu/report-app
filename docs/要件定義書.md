# 要件定義書：日報作成・Git自動連携ツール（仮）

## 1. システム概要
本システムは、エンジニア研修における日報作成およびGit運用を効率化・標準化するためのWebアプリケーションである。

- **用途**: 毎日の日報作成、Markdown生成、Gitリポジトリへの自動Push
- **対象ユーザー**: 研修対象者（新卒14名）

---

## 2. 業務要件
現状の課題（As-Is）を分析し、本システム導入後の理想の状態（To-Be）を定義する。

### 2.1 現状の課題 (As-Is)
- 14名全員のコミットメッセージやファイル構造、Markdownのフォーマットがバラバラである。
- 人事担当者が内容をチェックする際、該当ファイルを探す手間や確認の負荷が高い。
- 日報共有会において、各担当者が自分のファイルを見つけるのに時間がかかる。

### 2.2 導入後の状態 (To-Be)
- フォーマットの強制により、全員が統一されたファイル構造・Markdown形式で出力される。
- コミットメッセージが自動生成され、Git履歴の視認性が向上する。
- 構造が統一されることで、人事のチェック効率および振り返り時の検索性が向上する。

---

## 3. システム要件
業務要件を実現するために必要なシステム特性。

- **Markdown レビュアー**: 入力内容をリアルタイム（またはプレビューボタン）で確認できる機能。
- **記入フォーム**: 直感的に入力可能なWebフォーム。
- **ディレクトリ管理**: 各個人のディレクトリへ自動で振り分ける仕組み（保存先パスの自動解決）。
- **対象ビューポート**: PCメイン（研修中のPC作業を想定）。
- **SP対応**: 検討中（参照メインのレスポンシブ対応を視野に入れる）。

---

## 4. 機能要件
### 4.1 入力フォーム機能
以下の項目をユーザーが入力し、Markdownファイルを生成する。
- **名前**: 投稿者の識別（ディレクトリパスの解決に使用）。
- **日付**: 日報の対象日（ファイル名およびコミットメッセージに使用）。
- **個人ディレクトリへのパス**: （※運用ミス防止のため、名前から自動生成する仕組みを推奨）。
- **日報本文**: 自由記述のテキストエリア。

### 4.2 Git連携機能 (JGit実装)
- **自動プレフィックス**: コミットメッセージの先頭に `docs: ` を強制挿入。
- **メッセージ編集**: `docs: ` 以降のテキストはユーザーが任意に変更可能。
- **競合回避ロジック**:
  - Push実行直前に必ず `Fetch` & `Rebase` (またはPull) を実行。
  - 最新の状態をローカルに取り込んでからPushすることで、多人数同時利用時の競合を回避する。

---

## 5. 非機能要件
### 5.1 セキュリティ・環境
- **デプロイ先**: 研修資料等の機密情報が含まれるため、以下のいずれかを検討。
  - 案A: 社内ネットワーク内サーバー（VPN必須）
  - 案B: 認証（SSO/Basic認証）付きのPaaS環境
  - 案C: 各自のローカル実行（localhost）
- **可用性**: 研修期間中に安定して稼働すること。

---

## 6. 技術要件
### 6.1 技術スタック
- **バックエンド**: Java 21 / Spring Boot
- **フロントエンド**: TypeScript / React / HTML5 / CSS3
- **CSSライブラリ**: Tailwind CSS
- **Gitライブラリ**: JGit

---

## 7. 後対応（フェーズ2以降）
初期リリース後の追加機能として以下を検討する。

- **高度な下書き保存機能**:
  - **テキストデータ**: `localStorage` を使用し、ブラウザを閉じても内容を保持。
  - **画像データ**: `IndexedDB` を使用し、大容量の画像データもブラウザ内に一時保存可能にする。
